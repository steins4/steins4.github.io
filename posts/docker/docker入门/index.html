Docker入门

  1. 镜像和容器

    1. image镜像

      1. Docker image是一个read-only文件
      2. 这个文件包含文件系统，源码，库文件，依赖，工具等一些运行application所需要的文件
      3. docker image具有分层的概念

    2. container容器

      1. 一个运行中的dockerimage
      2. 实质是复制image并在image最上层加上一层read-write的层（container layer）
      3. 基于同一个image可以创建对歌container

    3. image镜像的获取

      1. 自己制作
      2. 从registry拉取

  2. 容器操作

    1. docker container run nginx:前台运行
    2. 停止容器
      1. docker container stop container_id/NAMES
    3. 查看正在运行的容器：docker container ls,查看正在运行和退出的容器docker container ls -a
    4. 删除容器：docker container rm CONTAINER_ID
    5. 查看所有正在运行的container_id:docker container ls -aq
    6. 批量停止所有的container：docker container stop $(docker container ls -aq)
    7. 批量删除所有的container：docker container rm $(docker container ls -aq)
    8. docker container run -d -p 80:80 nginx :创建后台运行的docker容器
    9. docker attach container_id :在终端前台可以看日志
    10. docker container logs container_id:查看日志
    11. docker container logs -f container_id:动态的跟踪日志
    12. docker exec -it container_id sh:进入容器内部并和容器进行交互

  3. 容器和虚拟机

    1. 容器不是虚拟机

      1. 容器是进程
      2. 容器中的进程被限制了对CPU内存等资源的访问
      3. 当进程停止后，容器就退出了

  4. docker container run背后发生了什么

    1. docker container run -d -p 80:80 nginx
      1. 在本地查找是否有nginx这个image镜像，但是没有发现
      2. 去远程的image registry查找image镜像（默认的registry是Docker Hub）
      3. 下载最新版本的nginx镜像（默认是image：latest）
      4. 基于下载镜像来创建一个新的容器，并且准备运行
      5. docker engine分配给这个容器一个虚拟ip地址
      6. 在宿主机上打开80端口并把容器的80端口转发到宿主机上
      7. 启动容器，运行指定的命令，（这里是一个shell脚本去启动nginx）

  5. 镜像

    1. 获取的三种方式

      1. pull from registry（online）从registry获取
        1. public（公有）
        2. private（私有）
      2. build from Dockerfile（online）从Dockerfile构建
      3. load from file（offline）文件导入：离线

    2. 操作

      1. 拉取镜像：docker image pull image_name
      2. 查看docker镜像：docker image ls
      3. 显示镜像详细信息：docker image inspect image_id
      4. 删除镜像：docker image rm image_id

    3. Dockerfile

      1. 介绍

        1. Dockerfile是用于构建docker镜像的文件
        2. Dockerfile里包含了构建镜像所需的指令
        3. Dockerfile有特定的语法规则
