  1. Buffer

    Buffer对象可以看做一个固定数量的数据的容器，其作用是一个存储器。缓冲区和通道紧密联系。通道是I/O传输发生时通过的入口，而缓冲区是数据的来源或者目标。

    1. 缓冲区基础

      1. 属性

        1. 容量：
          1. 缓冲区能够容纳的数据元素的最大数量，这一容量在缓冲区创建时被设定，并且永远不能被改变
        2. 上界
          1. 缓冲区的第一个不能被读或写的元素，或者说，缓冲区现存元素的计数
        3. 位置
          1. 下一个要被读或写的元素的索引，位置会自动由相应的get()和put()函数更新
        4. 标记
          1. 一个备忘位置，调用mark()来设定mark=position。调用reset()设定position=mark。标记在设定前是未定义的。

      2. 函数

        1. flip() 翻转函数

        2. compact()压缩函数
          1. 压缩前
          2. 压缩后
        3. equals()和compareTo()
          1. 两个缓冲区被认为相等的充要条件
            1. 两个对象类型相同，包含不同数据类型的buffer永远不会相等，而且buffer绝不会等于非buffer对象
            2. 两个对象都剩余同样数量的元素，Buffer的容量不需要相同，而且缓冲区中剩余数据的索引也不必相同，但每个缓冲区中剩余元素的数目（从位置到上界）必须相同
            3. 在每个缓冲区中应被Get（）函数返回的剩余数据元素序列必须一致
          2. compareTo函数是按照词典顺序进行比较的，在左边的缓冲区参数小于，等于或大于右边的缓冲区实，分别返回一个负整数，0和正整数。
        4. 批量移动
          1. get
            2. get(char[] dat)等价于get(char[] dat, 0, dat.length)
            3. 当传入的数组的长度大于缓冲区get的元素长度（即缓冲区的数据不够完全填满数组）时，会抛异常
          2. put
            2. 如果缓冲区有足够的空间接受数组中的数据，数据将会被复制到从当前位置开始的缓冲区。如果缓冲区没有足够的空间，那么不会有数据被传递，同时抛出一个BufferOverflowException异常
            3. 也可以通过调用带有一个缓冲区引用作为参数的put()来在两个缓冲区内进行批量传递
              1. buffer.put(srcBuffer)等价于
              2. 不能对一个缓冲区传递他自己
              3. srcRemaining如果大于dstRemaining，会抛出异常

    2. 创建缓冲区

      1. CharBuffer.allocate(100)

        1. 这段代码表示从推空间分配了一个char型数组来作为备份存储器来存储100个char变量

      2. CharBuffer.wrap(charArray)

        1. 需要自己提供一个数组来存储缓冲区数据

        2. char[] charArray = new char[100];
CharBuffer buffer = CharBuffer.wrap(charArray);

        3. 这意味着调用put方法会影响到charArray数组，对charArray数组的任何改变都会对这个缓冲区对象可见

      3. CharBuffer charBuffer = Charbuffer,wrap(myArray, 12, 42)

        1. 创建了一个position为12，limit为54，容量为myArray.length的缓冲区。

      4. 通过allocate和wrap函数创建的缓冲区都是间接的。间接的缓冲区使用备份数组.Boolean函数hasArray()告诉我们这个缓冲区是否有一个可存取的备份数组。如果这个函数的返回true，array()函数会返回这个缓冲区对象所使用的数组存储空间的引用。

    3. 复制缓冲区

      1. duplicate()
        1. 函数创建一个与原始缓冲区相似的新缓冲区，两个缓冲区共享数据元素
      2. asReadOnlyBufferException
        1. 函数可以生成一个只读的缓冲区视图。这个新的缓冲区不允许使用put()并且其isReadOnly()函数将会返回true
      3. slice()
        1. slice()分割函数和复制相似，但slice()创建一个从原始缓冲区的当前位置开始的新缓冲区，并且其容量是原始缓冲区的还是剩余元素数量。这个新缓冲区与原始缓冲区共享一段数据元素子序列，分割出来的缓冲区也会继承只读和直接属性和数组。

    4. 字节缓冲区

      1. 字节缓冲区

        1. 大端字节顺序（58700999）
        2. 小端字节顺序（58700999）

        3. Java的默认字节顺序是大端字节顺序

      2. 直接缓冲区

        1. 直接字节缓冲区通常是I/O操作最好的选择。在设计方面，他们支持JVM可用的最高效I/O机制。如果向一个通道中传递一个非直接ByteBuffer对象用于写入，通道可能会在每次调用中隐含地进行下面的操作
          1. 创建一个临时的直接ByteBuffer对象
          2. 将非直接缓冲区的内容复制到临时缓冲中
          3. 将非直接缓冲区执行低层次I/O操作
          4. 临时缓冲区对象离开作用域，并最终成为被回收的无用数据
        2. ByteBuffer.allocateDirect(int capacity);

      3. 视图缓冲区

        1. 每个视图对象维护他自己的属性，容量，位置，上界和标记，但是和原来的缓冲区共享数据元素。在ByteBuffer中允许创建视图来将byte型缓冲区字节数据映射为其它的原始数据类型。例如，adLongBuffer()函数创建一个将八个字节型数据当成一个long型数据来存取的视图缓冲区
